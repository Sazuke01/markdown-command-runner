# Principios de programación usados

## Principios SOLID

Principios para un software mantenible y buena estructura como legibilidad

### S

Single Responsibility Principle (Principio de Responsabilidad Única - SRP): Una clase debe tener una sola razón para cambiar, es decir, debe realizar una sola tarea o función específica.

### O

Open/Closed Principle (Principio de Abierto/Cerrado - OCP): Las entidades de software (clases, módulos, funciones) deben estar abiertas para la extensión, pero cerradas para la modificación.

### L

Liskov Substitution Principle (Principio de Sustitución de Liskov - LSP): Las subclases deben ser sustituibles por sus clases base sin alterar el correcto funcionamiento del sistema.

### I

- Interface Segregation Principle (Principio de Segregación de Interfaces - ISP): Es preferible tener muchas interfaces específicas que una sola interfaz de propósito general. Ningún cliente debe depender de métodos que no utiliza.

### D

- Dependency Inversion Principle (Principio de Inversión de Dependencia - DIP): Los módulos de alto nivel no deben depender de módulos de bajo nivel; ambos deben depender de abstracciones. Las dependencias deben estar en las interfaces, no en las clases concretas.
